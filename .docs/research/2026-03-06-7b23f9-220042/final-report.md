## 1. エグゼクティブサマリー

Qwen 3.5 27B のローカル導入は**技術的に実現可能**だが、**3つの構造的障壁**が確認された。

| 障壁 | 深刻度 | 概要 |
|------|--------|------|
| Ollama の Qwen3.5 JSON 互換性バグ | **Critical** | Issue #14493: フォーマット不一致（Hermes-style vs Qwen3-Coder XML）。エンドユーザーによる修正不可 |
| デプロイアーキテクチャの根本的変更 | **High** | Edge Runtime からローカル LLM への接続は構造的に不可能。Node.js Runtime への変更が必須 |
| 日本語占い文品質の不確実性 | **Medium** | GPT-4o 比で約9%劣後（ベンチマーク）。占い文固有の品質評価データは皆無 |

**推奨アクション**: 3ゲート方式による段階的導入。Gate 0（実機検証 4-8h）で全リスクの解像度を上げ、致命的問題があれば最小損失で撤退可能。

### 3.1 技術的実現性

> ※ この視点は調査途中で失敗したが、主要な知見は取得済み

**OpenAI 互換 API の互換性**

| 実行基盤 | chat.completions | streaming | json_object | json_schema |
|----------|-----------------|-----------|-------------|-------------|
| Ollama | Yes | Yes | Yes (format マッピング) | 既知バグあり (#10001) |
| LM Studio | Yes | Yes | Yes | Yes (正式サポート) |
| llama.cpp server | Yes | Yes (SSE) | Yes | バグ報告あり (#11847) |

- **streaming + json_object の同時使用**は3ツールとも明示的に保証されていない（グレーゾーン）
- Ollama の OpenAI 互換は **"experimental"** と明記されており、破壊的変更の可能性あり

**Qwen 3.5 27B のスペック**

| 項目 | 値 |
|------|------|
| IFEval（指示遵守） | 95.0（業界上位クラス） |
| コンテキスト長 | 262,144 トークン（YaRN 拡張で最大 1M） |
| 対応言語 | 201言語（日本語含む） |

**SSE ストリーミング**: Ollama の `/v1/chat/completions` は OpenAI SSE プロトコルに準拠（`data:` プレフィックス、`data: [DONE]` 終端、`choices[].delta.content` 構造）。既存の partial-json パーサーはそのまま動作する見込み。

### 3.3 リスク分析

| ID | リスク | 深刻度 | 信頼度 | 緩和策 |
|----|--------|--------|--------|--------|
| R1 | 日本語占い文品質の劣後 | Medium | Medium | ブラインド比較テスト + プロンプト最適化 |
| **R2** | **JSON 構造化出力バグ (#14493)** | **Critical** | **High** | grammar-constrained decoding / llama.cpp 直接利用 |
| R3 | コンテキスト長超過 | Low | High | 262K トークンで 10枚占いは余裕（<10K トークン） |
| **R4** | **Ollama の VRAM リーク・クラッシュ** | **High** | **High** | 定期再起動 (systemd timer) / Docker restart policy |
| R5 | モデル非推奨化 | Low | Medium | Apache 2.0 のため自己ホスト継続可能 |
| **R6** | **Vercel から LLM 接続不可** | **High** | **High** | セルフホスト移行 / トンネリング |

**最重大リスク詳細: R2 — Ollama + Qwen3.5 JSON バグ**

- Ollama は Qwen3 用に Hermes-style JSON フォーマットを使用
- Qwen3.5 は Qwen3-Coder XML フォーマット（`<function>` タグ）で訓練済み
- このフォーマット不一致により、JSON/ツール呼び出しが破損する
- `repeat_penalty` / `presence_penalty` が Go ランナーでサイレントに無視される
- **エンドユーザーによる修正は不可能**（Ollama コードベース側の修正待ち）

### 3.6 デプロイ構成

**3つのアーキテクチャパターン**

| パターン | メリット | デメリット | 推奨度 |
|---------|---------|----------|--------|
| (a) ローカル開発専用 (localhost) | 最もシンプル・安全 | 単一ユーザー前提 | Phase 1 向け |
| (b) Vercel + トンネリング | 既存デプロイ維持 | 遅延・可用性・body 制限 | 非推奨 |
| **(c) Docker Compose セルフホスト** | **レイテンシ最小・プライバシー最大** | インフラ管理コスト | **本番向け推奨** |

**重要**: Edge Runtime は Vercel 上でローカルサービスへ直接接続できない。LLM プロキシには必ず **Node.js Runtime** の API Routes を使う必要がある。

## 5. 推奨アクション

### 5.1 プライマリ: 3ゲート方式による段階的導入

```
Gate 0 (実機検証)  ──GO──>  Gate 1 (接続実装)  ──GO──>  Gate 2 (統合検証)
   4-8h                        3-5日                      1-2日
```

#### Gate 0: 実機検証（4-8時間）

Ollama をインストールし、Qwen 3.5 27B Q4_K_M を pull して以下を実測:

| 検証項目 | 内容 | GO 条件 | ABORT 条件 |
|----------|------|---------|-----------|
| (a) JSON 安定性 | Issue #14493 の再現性検証 + GBNF 回避テスト | 3層防御で安定処理可能 | リトライ 5回超でも不安定 |
| (b) 占い文品質 | 既存プロンプトでブラインド比較（各3サンプル） | GPT-4 比 80% 以上 | GPT-4 比 50% 未満 |
| (c) 速度 | TTFT + tok/s 実測 | TTFT<3秒, tok/s>15 | - |
| (d) VRAM | nvidia-smi 実測 | 24GB 以内 | - |

#### Gate 1: 接続レイヤー実装（3-5日）

Gate 0 GO 後のみ実施:

1. `client.ts` のベース URL / モデル名を環境変数化（`OLLAMA_BASE_URL`, `LLM_MODEL`）
2. Edge Runtime → Node.js Runtime への変更（占い生成 API ルートのみ）
3. フォールバック実装（Ollama → OpenAI API 自動切替）
4. 同時リクエスト制御（`OLLAMA_NUM_PARALLEL=1` + リクエストキュー）
5. プロンプト調整の適用
6. テストの接続先変更・モック更新

#### Gate 2: 統合検証（1-2日）

1. 全4スプレッド x ストリーミング動作の E2E 検証
2. ケルト十字10枚の段階的ストリーミング完走確認
3. フォールバック動作確認（Ollama 停止 → OpenAI 自動切替）
4. コールドスタート / ウォームスタートの UX レイテンシ測定

### 5.2 フォールバック: llama.cpp server 直接利用

**発動条件**: Gate 0(a) で Ollama 上の JSON 出力が安定しない場合、または Issue #14493 の修正が2週間以上見込めない場合

- llama.cpp server は OpenAI 互換度が最も高く、JSON mode をネイティブサポート
- Ollama の Qwen3.5 テンプレート問題を回避できる可能性あり
- Open WebUI を llama.cpp server に接続して運用性を確保

### 5.3 ABORT 条件

以下のいずれかで中止:

1. 品質が GPT-4 比 50% 未満かつプロンプト最適化でも改善しない
2. Ollama・llama.cpp 双方で JSON エラー率 > 30%（4層防御適用後）
3. GPU が VRAM 要件（17GB）を満たさず、購入が不承認

**ABORT 時の機会コスト**: OpenAI API 費用の継続（月 $45-90）、データプライバシー未改善、オフライン動作不可。ただし既存設計（d-20260304-005644）は完成度が高く機能的問題なし。Gate 0 の投資損失は最大 4-8h。
