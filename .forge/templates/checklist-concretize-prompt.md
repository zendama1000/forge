## タスク

上記の human_check 確認項目（レベルA）を、実装済みコードを読んだ上で、レベルB（操作手順付きchecklist）に具体化してください。

### 手順

1. 作業ディレクトリ内の実装コード（ルーティング、コンポーネント、API定義等）を読む
2. 各 human_check 項目について、実際のURL・ボタン名・入力欄名・APIエンドポイントを特定する
3. 人間が画面を見ながら1ステップずつ実行できる操作手順に変換する
4. 各ステップに「操作」と「期待結果」を含める

### 出力形式

Markdown形式のチェックリストを出力してください。

```
# {{DEV_PHASE_ID}} 目視確認チェックリスト

## 前提条件
- サーバー起動済み: [実際のURL]
- その他の前提（あれば）

## 確認手順

### 1. [確認項目名]（元のhuman_check項目に対応）
- [ ] [操作] → [期待結果]
- [ ] [操作] → [期待結果]
```

### 注意事項

- 実装コードに存在する機能のみを対象とすること。実装されていない機能の手順は生成しない
- URLやボタン名はコードから読み取った実際の値を使うこと
- コードフェンスは出力に含めないこと。Markdownテキストをそのまま出力する
